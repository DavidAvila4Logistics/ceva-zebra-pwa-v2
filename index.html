Cómo usar CEVA Zebra v2 (sin backend, 100% browser)

Esta versión funciona completamente en el navegador — no necesita servidor, Node.js ni scripts .bat.

  • El HR recomendado es XLSX. Exportá el romaneo desde el sistema como Excel antes de cargarlo aquí. (El PDF no está soportado en esta versión estática).
  • Cargá uno o varios archivos Impresos XLSX (pueden ser múltiples días).
  • Cargá el archivo Ce.xlsx o JSON con la tabla Localidad → Ruta.
  • Presioná Procesar — el resultado se descarga automáticamente como XLSX.

Instrucciones y Lógica de Procesamiento

Formato del HR XLSX

El HR debe tener columnas: Nº Factura, Destinatario, Ciudad Destinatario. La detección es flexible — se normalizan nombres de columnas (sin tildes, minúsculas). Si el sistema genera PDF, primero exportalo como XLSX desde Excel o el propio sistema.

Formato de Impresos XLSX

Deben tener columnas Remito y Handling Unit (o variantes: Referencia, HU, Unidad). Se aceptan múltiples archivos — se concatenan automáticamente.

Derivar Nº Factura desde Remito

Ej: 0300-00018664 → 18664 (se quita el prefijo antes del guion y los ceros a la izquierda).

Merge y deduplicación

1. Impresos → derivan Nº Factura. 2. HR → lookup por Nº Factura para obtener Destinatario y Ciudad. 3. Ce → lookup por Ciudad normalizada para obtener Ruta. 4. Se desduplicada por Handling Unit. 5. Se generan alertas si hay facturas en HR sin match en Impresos o viceversa.

Ce.xlsx formato

Primera hoja con columnas Localidad y Ruta. Alternativa: JSON array con [{"localidad":"CORDOBA","ruta":"R1"}]
